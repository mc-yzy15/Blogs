# 腾讯开悟深度测评：一场硬件与软件的"套娃"噩梦，普通开发者的绝望指南

朋友们，你们用过腾讯开悟吗？

没用过？那恭喜你，你是个正常人。你的头发还在，你的钱包还在，你的san值还是满的。

用了？那咱们握个手吧，同是天涯沦落人，相逢何必曾相识。你的头发少了多少？钱包瘪了多少？san值还剩几点？

我算是看明白了，腾讯开悟这玩意儿，不是给普通人用的，是给那些家里有矿、手上有卡、还闲得没事干折腾环境的人用的。你要是跟我一样，就想搞个AI集群训练，那你可真是来对地方了——来受罪的地方了。

这篇文章，我要把腾讯开悟的每一层套娃都扒开给你看，让你知道这玩意儿到底有多离谱。

---

## 前言：腾讯开悟是什么？为什么这么难用？

先给不了解的朋友科普一下：**腾讯开悟**（AI Arena）是腾讯牵头构建的AI多智能体与复杂决策开放研究平台，依托腾讯AI平台部和「王者荣耀」在算法、算力、实验场景方面的核心优势，为学术研究人员和算法开发者提供研究环境。简单来说，就是让你训练王者荣耀AI的一个平台。

听起来很美好是吧？腾讯出品，王者荣耀加持，AI研究平台，多高大上啊！

但等你真正开始用的时候，你就会发现：这玩意儿的门槛，比珠穆朗玛峰还高。

为什么？因为腾讯开悟的架构设计，从根本上就存在一个巨大的矛盾：**它要求你同时拥有Windows环境和Linux环境，还要求你有高端GPU，还要求你支持嵌套虚拟化，还要求你在Docker里跑腾讯自己的Linux发行版。**

这每一个要求单独拿出来都不算过分，但组合在一起，就是一个不可能完成的任务。就像有人跟你说："你只需要会做饭、会修车、会做手术、会开飞机就行了。"——每个技能都不难学，但你要一个人同时会四样？你当我是达芬奇啊？

好，废话不多说，让我一层一层给你扒开这个套娃。

---

## 第一层套娃：Windows——2026年了还只支持Windows？

### gamecore只支持Windows，这是原罪

腾讯开悟的核心组件之一是**gamecore**（游戏引擎），它负责模拟王者荣耀的游戏环境。而gamecore，**只支持Windows系统**。

这不是我瞎说，腾讯开悟官网白纸黑字写着："仅支持Windows系统"。你去申请gamecore的时候，登录后就能看到这个提示。

你说你用Linux？不好意思，gamecore是Windows程序，`.exe`文件，你Linux跑不了。虽然官方在GitHub上提供了Wine方案（`run_windows_gamecore_on_linux.md`），可以在Linux上通过Wine跑gamecore，但那兼容性简直一言难尽。我看过不少开发者的反馈，各种奇奇怪怪的bug层出不穷：端口连不上、画面渲染异常、内存泄漏、跑着跑着就崩了……训练到一半数据全丢，几天的训练白费。

官方自己都推荐Windows优先，Wine方案就是个摆设，一个"我们有Linux支持"的面子工程。

你说你用macOS？更别想了，连个Wine方案都没有，直接把你拒之门外。苹果用户出门右转，不送。

### 这是什么逻辑？

我就纳了闷了，现在都2026年了，AI训练的主流平台是Linux，这是业界共识。PyTorch官方推荐Linux，TensorFlow官方推荐Linux，Hugging Face官方推荐Linux，就连你们腾讯自己家的TencentOS都是Linux发行版！你们腾讯内部的服务器跑的也是Linux啊！

结果你们的开悟平台gamecore居然只支持Windows？

这是什么逻辑？你们自己都不用Windows训练，凭什么让我们用？

这就好比你开了一家面馆，自己员工吃米饭，却逼着顾客吃面条。你说这叫什么事儿？

### 为什么不做Linux版？

我理解，gamecore可能依赖了Windows的某些底层API，迁移到Linux有技术难度。但问题是：

1. 你们腾讯自己家有TencentOS，有Linux技术积累，做个Linux版gamecore很难吗？
2. Wine方案你们都做了，说明你们知道Linux用户有需求，那为什么不直接做原生Linux版？
3. 你们的集群训练方案里，gamecore最终也是跑在Linux服务器上的（通过Wine），那为什么不直接做原生支持？

答案很简单：**不想做**。或者说，**优先级不够**。腾讯开悟是一个学术研究平台，不是商业产品，腾讯没有动力去优化用户体验。反正你们爱用不用，比赛照样有人参加，论文照样有人发。

但这对普通开发者来说，就是一堵翻不过去的墙。

## 第二层套娃：嵌套虚拟化——云服务器的死穴

### WSL2和Docker都依赖虚拟化

腾讯开悟的架构是这样的：gamecore跑在Windows上，hok_env（AI训练SDK）跑在Linux上。怎么在Windows上跑Linux？官方推荐两种方案：

1. **WSL2**（Windows Subsystem for Linux Version 2）
2. **Docker Desktop for Windows**

但问题是，这两个方案都依赖虚拟化技术。

WSL2本质上就是一个轻量级的Hyper-V虚拟机，它运行在一个完整的Linux内核上。Docker Desktop for Windows也依赖WSL2后端（新版Docker Desktop已经不再使用Hyper-V，而是直接使用WSL2）。

也就是说，不管你选哪条路，你都需要在Windows上跑虚拟机。这本身不是问题，大部分个人电脑的CPU都支持虚拟化（Intel VT-x或AMD-V），在BIOS里开启就行。

### 云服务器不支持嵌套虚拟化——这才是致命的

但如果你想在**云服务器**上跑呢？兄弟，这才是真正的坑。

腾讯云官方文档白纸黑字写着：

> **Windows Subsystem for Linux（简称 WSL）在 Windows Server 2022 系统中遇到了一些限制。由于系统内核完整性的原因，WSL 1无法运行 Linux Docker。虽然 WSL 2 可以运行 Linux Docker，但它需要硬件支持二次虚拟化，而普通的云服务器（包括 Lighthouse）不支持二次虚拟化。**

翻译成人话就是：你花大价钱租的普通Windows云服务器，开不了WSL2，也就跑不了Docker，也就用不了腾讯开悟。

为什么？因为云服务器本身就是虚拟机（你的Windows是跑在宿主机的Hypervisor上的），而WSL2又需要在Windows里再开一个虚拟机（Hyper-V），这就是"虚拟化里面再虚拟化"——也就是所谓的**嵌套虚拟化**（Nested Virtualization）。大部分云服务商的普通虚拟机都不支持这个功能，因为开启嵌套虚拟化会带来性能损耗和安全风险。

### 裸金属？贵得离谱

那什么服务器才支持嵌套虚拟化？裸金属服务器（Bare Metal Server）。裸金属就是物理服务器，没有虚拟化层，你可以随便装系统、开虚拟机。

但裸金属多少钱？腾讯云带GPU的裸金属服务器，比如HCCPNV4h（8卡A100），一个月好几万起步。而且还不一定有Windows系统可选，腾讯云的裸金属大部分只提供Linux镜像。就算有Windows，你还得自己装驱动、装Docker、配环境，折腾一个星期起步。

我就想问问腾讯：**你们自己家的云服务器不支持你们自己家软件需要的功能，你们不觉得尴尬吗？**

这就好比你开了一家餐厅，结果你们自己的厨房做不出你们菜单上的菜，你让顾客怎么想？顾客点了菜，你说"不好意思，我们的厨房不支持做这道菜，您得自己带锅碗瓢盆来我们这儿做"——你说这不是胡闹吗？

### 其他云平台也差不多

你可能会想，那我去阿里云、华为云、AWS租呢？抱歉，情况差不多。所有主流云服务商的普通GPU云服务器都不支持嵌套虚拟化。AWS的裸金属实例倒是可以，但价格更贵。而且就算你找到了支持嵌套虚拟化的云服务器，还得是Windows系统 + 高端GPU + 支持嵌套虚拟化，这三个条件同时满足的云服务器，市面上几乎不存在。

## 第三层套娃：Docker + TencentOS——套中套，套外还有套

### Docker镜像基于TencentOS

好不容易搞定了虚拟化的问题，接下来你还得装Docker。腾讯开悟提供了Docker镜像，分CPU版和GPU版：

- CPU版：`tencentailab/hok_env:cpu_v2.0.1`
- GPU版：`tencentailab/hok_env:gpu_v2.0.1`

GPU版需要NVIDIA显卡支持，还得装nvidia-docker（NVIDIA Container Toolkit）。

而且这个Docker镜像是基于**TencentOS**的——对，腾讯自己搞了个Linux发行版叫TencentOS Server（又名Tencent Linux，简称Tlinux），是腾讯针对云场景研发的Linux操作系统。这系统本身没毛病，在腾讯内部占比超99%，覆盖了腾讯所有业务，稳定性没问题。腾讯云上的很多实例默认就是TencentOS，Docker也预装好了，开箱即用。

但问题是——你让我直接装TencentOS不行吗？不行！腾讯开悟的官方方案就是让你在Docker里跑TencentOS镜像，不给你别的选择。

### 八层套娃，俄罗斯套娃都没你们能套

你想想这个套娃有多离谱：

1. **第一层**：你得有一台Windows电脑或服务器
2. **第二层**：Windows上开启Hyper-V虚拟化
3. **第三层**：Hyper-V里跑WSL2（一个轻量级Linux虚拟机）
4. **第四层**：WSL2里跑Ubuntu发行版
5. **第五层**：Ubuntu里装Docker引擎
6. **第六层**：Docker里跑TencentOS容器镜像
7. **第七层**：TencentOS里跑hok_env SDK
8. **第八层**：hok_env通过网络连接Windows上的gamecore

八层！八层套娃！俄罗斯套娃都没你们能套！

我打个比方：你想吃碗面，结果老板告诉你，你得先买个碗，碗里面放个锅，锅里面放个灶，灶上面再放个电磁炉，电磁炉上面再放个砂锅，砂锅里面才能煮面。你直接给我个砂锅不行吗？不行，必须全套。

### 脱裤子放屁的架构设计

而且最离谱的是什么？TencentOS本身就是个完整的Linux操作系统，你完全可以把它装在裸金属服务器上直接用。但腾讯偏不，非要让你在Windows里面套WSL2，WSL2里面套Docker，Docker里面再套TencentOS。

这不是脱裤子放屁是什么？

你说你直接让我装TencentOS都行啊，我装个双系统怎么了？我直接买台服务器装TencentOS跑训练不就完了吗？

不行，因为gamecore只支持Windows。那gamecore你倒是做个Linux版啊？也不做。那Wine方案你倒是好好维护啊？也不维护。

就这？就这？

### 性能损耗：套一层慢一层

而且每套一层虚拟化，就有性能损耗。WSL2虽然有微软的优化，但毕竟还是虚拟机，IO性能、网络性能都有损耗。Docker虽然是容器化，比虚拟机轻量，但在WSL2里跑Docker，等于是在虚拟机里跑容器，性能损耗是叠加的。

你花大价钱租的GPU服务器，结果因为套了这么多层虚拟化，GPU利用率大打折扣。本来能跑100%的算力，套了三层之后可能只剩80%了。你租一个月几万块的服务器，20%的算力白白浪费在虚拟化开销上，你说亏不亏？

## 第四层套娃：显卡——为了这盘醋，包了整锅饺子

### 本地测试vs集群训练

如果你只是本地跑个1v1测试，那随便一张NVIDIA显卡就行，甚至CPU版都能凑合。腾讯开悟的Docker镜像提供了CPU版本，不需要GPU也能跑，只是速度慢得像蜗牛。

但如果你想认真搞AI集群训练呢？那就得上专业计算卡了。

腾讯开悟官方用的就是V100，NVIDIA的博客里明确提到了"腾讯开悟AI开放研究平台选择使用NVIDIA V100 GPU和NVIDIA TensorRT推理引擎来加速推理"。腾讯云上开悟相关的GPU实例也提供A10、V100、A100三种选择。

### 显卡价格详解

那这些卡什么价？我给你算算，让你死个明白：

**V100（NVIDIA Tesla V100）**：

这是腾讯开悟官方推荐的基础卡。淘宝上二手的V100 SXM2 16G，单独显卡芯片大概 **500-800块**，但这是没有散热和转接板的"裸卡"——就是一个光秃秃的PCB板，连散热器都没有。要想在普通电脑上用，还得买：

- 涡轮散热器：300-500块
- SXM2转PCIe转接板：300-800块
- 电源线、转接线等杂七杂八：100-200块

整套下来 **1500-3000块** 跑不掉。而且还是二手拆机货，品控全靠运气，买回来能不能点亮都是个问题。有些卖家卖的是翻新卡，外观看着挺新，实际上核心已经老化了，跑训练动不动就报CUDA错误。

V100 32GB的版本更贵，闲鱼上大概 **2000-3700块** 不等，而且价格还在涨，因为V100虽然停产了，但需求还在，供不应求。

你要是买全新的？不好意思，V100早就停产了，市面上全是二手拆机货和翻新卡。你花几千块买一张不知道转了几手的显卡，心里能踏实吗？

**A10（NVIDIA A10）**：

这是腾讯云PNV4实例搭载的显卡，24GB显存，定位中端。个人买不到，只能租。腾讯云A10单卡包月大概 **4000-5000元**。

一个月五千，一年六万。你算算这钱够买多少张RTX 4090了？一张4090大概12000-15000，六万块够买四张4090了。四张4090插在本地电脑上跑PyTorch，不比租一张A10香？

**A100（NVIDIA A100）**：

这是目前最顶级的AI训练卡之一。企业采购一片80GB的版本要 **几万到几十万不等**，个人根本买不到——你有钱也买不到，NVIDIA不卖给你，A100是面向企业的，需要企业资质和NVIDIA的审批。

算力租赁的话，A100大概 **4-6.5元/小时**，包月 **9800-30000元**。腾讯云A100单卡包月大概 **10000元左右**，8卡服务器更贵，一个月好几万。你租一年就是几十万，这钱够买辆不错的车了。

### 腾讯云GPU服务器价格

腾讯云自己的GPU服务器呢？

- 单卡V100（GN10Xp，1卡）：大概 **19元/小时**
- 8卡V100（GN10Xp，8卡）：大概 **170元/小时**
- 单卡A100（GT4，1卡）：大概 **30-40元/小时**
- 8卡A100（GT4，8卡）：大概 **250-300元/小时**

你跑一天8卡V100就是四千块，跑一个月就是十二万。跑8卡A100更贵，一个月可能要二十万。

但别忘了最关键的问题——**腾讯云的普通GPU云服务器不支持二次虚拟化**，你租了也跑不了WSL2，跑不了WSL2就用不了Docker，用不了Docker就跑不了腾讯开悟。

你花了几千几万租了GPU服务器，结果连个Docker都装不上。你找腾讯云客服，客服说"我们不支持嵌套虚拟化"；你找腾讯开悟客服，客服说"我们推荐用WSL2"。你两头跑，两头都不管你，你就是那个被踢来踢去的皮球。

**死循环了属于是。**

### 对比：PyTorch + RTX 4090

与其折腾腾讯开悟，不如看看隔壁PyTorch的体验：

- 买一张RTX 4090：12000-15000元
- 装好CUDA驱动：10分钟
- `pip install torch`：1分钟
- 开始训练：立刻

三步搞定，开箱即用。你腾讯开悟呢？八步起步，每一步都是坑。

## 现实困境：四条路，条条死路

说了这么多，可能有朋友要问了：你就不能老老实实用人家的标准配置吗？

标准配置？兄弟们，我跟你们讲，腾讯开悟的集群训练环境要求是这样的：
- Windows 10/11（gamecore必须跑在Windows上）
- 支持嵌套虚拟化的CPU（跑WSL2/Docker需要）
- WSL2或Docker Desktop（hok_env需要Linux环境）
- 基于TencentOS的Docker镜像（官方推荐）
- NVIDIA专业计算卡：A10/V100/A100（集群训练需要）

看起来每一条都不过分，但组合在一起就是灾难。我给你分析四条路，条条死路：

### 第一条路：云服务器——租了也白租

普通GPU云服务器不支持二次虚拟化，开不了WSL2，跑不了Docker。你花大价钱租了A100的GPU服务器，结果连Docker都装不上，更别提跑腾讯开悟了。

钱花了，事没办成，你说气不气？

而且你租GPU服务器是按小时计费的，你一边租着一边折腾环境，折腾一天就是几百上千块。等你环境配好了，钱也花得差不多了，训练还没开始呢。

### 第二条路：裸金属——贵得离谱还折腾

裸金属支持二次虚拟化，但腾讯云带GPU的裸金属服务器（比如HCCPNV4h，8卡A100）一个月好几万，而且不一定有Windows系统可选。腾讯云的裸金属大部分只提供Linux镜像，你要Windows还得自己装。

就算有Windows，你还得自己装GPU驱动、装Docker、装nvidia-docker、配WSL2、配网络、配环境变量……折腾一个星期起步。而且裸金属服务器不是随时有的，你得排队等，等个三五天是常事。

等你终于把环境配好了，一个月已经过去了，几万块也花出去了。

### 第三条路：本地跑——门槛高得吓人

你得有一台Windows电脑 + NVIDIA专业计算卡 + 支持虚拟化的CPU。V100整套二手1500-3000，但那是二手拆机货，能不能用全看脸。A100个人买不到。

然后还得折腾WSL2和Docker的环境配置，光是环境配置就能折腾好几天。WSL2的网络配置、Docker的GPU直通、nvidia-docker的版本兼容……每一个都是坑。

而且你一张卡能训练出什么来？集群训练至少得4卡8卡起步，你总不能买8张V100吧？那也得一万多，还得有主板支持8卡（普通主板根本插不了8张卡），还得有大功率电源（8张V100满载功耗2000W+），还得有散热（8张卡的热量能把房间变成桑拿房）……

### 第四条路：Linux服务器——Wine方案一言难尽

可以，但gamecore是Windows程序，你得用Wine来跑，又是一堆兼容性问题。

Wine是一个在Linux上运行Windows程序的兼容层，但它的兼容性远不如原生Windows。腾讯开悟的gamecore依赖了Windows的很多底层API，Wine不一定能完美兼容。而且Wine跑gamecore的稳定性远不如原生Windows，训练到一半崩了你找谁哭去？几天的训练数据全没了，你重新跑？再跑几天又崩了？

我看过不少开发者的反馈，Wine方案就是个坑，能用但不好用，随时可能出问题。对于需要长时间运行的集群训练来说，稳定性是第一位的，Wine方案根本不靠谱。

## 深度分析：腾讯开悟为什么这么难用？

### 根本原因：架构设计的先天缺陷

腾讯开悟之所以这么难用，根本原因在于它的架构设计存在先天缺陷：**gamecore和hok_env被强行拆分到了两个不同的操作系统上。**

gamecore是Windows程序，hok_env是Linux程序，两者通过网络通信。这种跨平台架构本身并不罕见，很多分布式系统都是这样的。但问题在于，腾讯开悟把这两个组件的部署责任全部推给了用户，而且没有提供任何简化的部署方案。

你想想，如果腾讯开悟提供以下任何一种方案，用户体验都会好很多：

1. **Linux版gamecore**：直接在Linux服务器上跑gamecore，不需要Windows，不需要WSL2，不需要嵌套虚拟化。一条路搞定。
2. **一体化Docker镜像**：把gamecore（通过Wine）和hok_env打包在同一个Docker镜像里，一条命令启动，不需要折腾Windows和WSL2。
3. **云端一体化服务**：直接在腾讯云上提供开悟的SaaS服务，用户不需要自己部署环境，登录就能用。
4. **裸金属一键部署**：提供带Windows + GPU + Docker + 开悟环境的裸金属镜像，一键开通，开箱即用。

但腾讯一个都没做。

### 为什么不做？因为不赚钱

腾讯开悟是一个学术研究平台，不是商业产品。腾讯没有义务为普通开发者优化用户体验，也没有动力去做这些事情。反正比赛照样有人参加，论文照样有人发，腾讯的目的已经达到了。

但对于我们普通开发者来说，这就是一场噩梦。我们不是腾讯的员工，没有腾讯内部的算力资源和技术支持，我们只有一台电脑和一颗想学AI的心。但腾讯开悟的门槛，把这颗心给磨没了。

### 对比：其他AI平台是怎么做的？

看看其他AI平台是怎么做的：

- **OpenAI Gym**：`pip install gym`，一行命令搞定。不需要Windows，不需要WSL2，不需要Docker，不需要专业计算卡。任何一台电脑都能跑。
- **DeepMind OpenSpiel**：`pip install open-spiel`，一行命令搞定。跨平台支持，Linux/macOS/Windows都行。
- **Facebook ELF**：开源的AI研究平台，支持Linux，Docker一键部署，不需要Windows。
- **微软Malmö**：基于Minecraft的AI平台，跨平台支持，Java写的，哪里都能跑。

这些平台都做到了一件事：**让用户把时间花在研究上，而不是花在环境配置上。**

腾讯开悟呢？你花80%的时间配置环境，20%的时间做研究。本末倒置了属于是。
